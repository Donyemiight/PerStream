Switch to Live Mode (Arc testnet)

PerStream runs in mock mode by default — every per-second tick is recorded but no real USDC moves. In live mode, ticks become real on-chain transactions on Arc testnet, verifiable on Arcscan.

✅ Already in live mode! The backend is connected to Arc testnet. All ticks are real on-chain transactions.

1 Get testnet USDC from the Circle Faucet

Visit the Circle Faucet and request 20 USDC on Arc Testnet:

Open Circle Faucet →
  1. Select "USDC"
  2. Select "Arc Testnet" from the network dropdown
  3. Paste the seller address (shown in step 2 below)
  4. Click "Send 20 USDC"
  5. Wait ~30 seconds for the transaction to confirm

The faucet is rate-limited to 20 USDC every 2 hours per address. The seller address below is the one the backend uses to receive on-chain payments.

2 Get the seller address

The backend's settlement wallet is derived from SETTLEMENT_PRIVATE_KEY in backend/.env. Get its current address:

curl -s /api/audit/stats | python3 -m json.tool

The sellerAddress field is the address you paste into the faucet.

3 Activate live mode on the backend

Once the faucet has sent USDC to the seller address, the backend owner (the hackathon submitter) needs to:

ssh user@perstream-host
cd /workspace/perstream
# Edit backend/.env and set:
#   PAYMENTS_MODE=live
#   SETTLEMENT_PRIVATE_KEY=0x_your_funded_key
pkill -f "node src/server"
cd backend
nohup env PAYMENTS_MODE=live PORT=3000 node src/server.js &

Refresh this page — if "Already in live mode!" appears at the top, you're done.

4 Verify on Arcscan

After the backend is in live mode, every per-second tick produces a real on-chain transaction. To verify:

  1. Open listen.html, sign in, pick a track, deposit, and start streaming
  2. Open creator.html, sign in as the demo creator
  3. Scroll to "🔍 On-chain audit trail"
  4. Each tick now has a clickable Arcscan link — click it to see the real transaction on Arc

You can also view the seller wallet directly:

View seller on Arcscan →
